1. Create a dedicated GitHub repo (or use your current app repo) for all bug tickets.
2. Create these labels in GitHub: source:form, needs-triage, ready, in-progress, blocked, resolved, duplicate, wont-fix, cannot-repro, P1, P2, P3.
3. Create a GitHub Project board with columns: New, Ready, In Progress, Blocked, Done.
4. Build a Google Form with fields: Short title, What happened, Expected behavior, Steps to reproduce, Urgency, Can reproduce?, Screenshot/link, Course/class.
5. Link the form to a Google Sheet (Responses).
6. Create a fine-grained GitHub personal access token with Issues: Read and Write for this repo only.
7. In the Google Sheet, open Extensions -> Apps Script.
8. Add a script that runs on form submit and creates a GitHub issue via API using form answers.
9. In that script, set initial labels on every new issue: source:form and needs-triage.
10. Store GITHUB_TOKEN, OWNER, and REPO in Apps Script project properties (not hardcoded).
11. Add an installable trigger in Apps Script: function = your submit handler, event = On form submit.
12. Submit one test response and confirm it creates a GitHub issue correctly.
13. Add a GitHub Action (.github/workflows/triage.yml) triggered on issues.opened to auto-assign priority (P1/P2/P3) and move to ready when enough info exists.
14. Define eligibility rule: Antigravity can only take issues labeled ready and not labeled in-progress.
15. Install GitHub CLI on the machine running Antigravity and run gh auth login.
16. Create scripts/claim-next-ticket.sh that queries issues with: open + ready + no in-progress, sorted by created desc (latest first), then claims one by adding in-progress, assigning, and commenting.
17. Create scripts/start-antigravity.sh that runs claim-next-ticket.sh first, then launches Antigravity with that issue URL/number in the startup prompt/context.
18. Replace your normal Antigravity launch command with start-antigravity.sh (alias/shortcut).
19. Require every fix PR to include Closes #<issue-number> so merge auto-closes the ticket.
20. Add a merge workflow (.github/workflows/resolve.yml) to remove in-progress and add resolved on linked issue closure.
21. Define close policy: if not fixed, close only with one reason label (duplicate, wont-fix, cannot-repro, or blocked).
22. Run a full end-to-end test: submit form -> issue created -> launch Antigravity -> issue claimed -> PR merged -> issue closed/labeled.
23. Operational rule: always launch Antigravity through the wrapper script so it always grabs the latest eligible ticket.
